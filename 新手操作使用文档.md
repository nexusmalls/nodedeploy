# Nexus 节点新手操作指南

本文档适合第一次部署 Nexus 验证者节点的用户。默认流程是：**在本机编辑配置 → 一键同步到服务器 → 服务器后台部署 → 查看日志和状态**。

## 一、你需要准备什么

### 1. 本机

本机需要有这个目录：

```bash
/home/XXXX/桌面/nodedeploy
```

目录里至少包含：

```text
remote_deploy.sh
部署入口，本机执行

deploy_nexus_server.sh
真正的服务器部署脚本

deploy.env
你的部署配置

mainnet-raw.json
链配置文件
```

### 2. 服务器

建议使用 Ubuntu 服务器，并使用 `root` 登录执行部署。
部署脚本需要 root 安装依赖、写 systemd、配置 Nginx/Certbot；公网 RPC 进程会自动以低权限账户运行。

服务器建议配置：

```text
内存：8GB 或以上更好
磁盘：20GB 以上可用空间
系统：Ubuntu
用户：root
```

### 3. 必要信息

部署前需要准备：

```text
服务器 IP
Bootnode 地址
验证者助记词
验证者地址，X 开头
```

如果你要部署 RPC 节点，还需要：

```text
RPC 域名
邮箱，用于申请 HTTPS 证书
```

当前默认只部署验证者节点。

## 二、编辑配置文件

进入本机部署目录：

```bash
cd /home/XXXX/桌面/nodedeploy
```

编辑配置：

```bash
nano deploy.env
```

重点检查这些配置。

### 1. 远程服务器配置

```bash
SERVER_HOST="你的服务器IP"
SERVER_USER="root"
SERVER_PORT="22"
SERVER_DIR="/root/nodedeploy"
```

示例：

```bash
SERVER_HOST="1.2.3.4"
SERVER_USER="root"
SERVER_PORT="22"
SERVER_DIR="/root/nodedeploy"
```

### 2. 部署模式

默认只部署验证者：

```bash
DEPLOY_MODE="validator"
```

可选值：

```bash
DEPLOY_MODE="validator"  # 只部署验证者节点，默认推荐
DEPLOY_MODE="rpc"        # 只部署 RPC 节点
DEPLOY_MODE="both"       # 同时部署验证者 + RPC
```

新手建议先保持：

```bash
DEPLOY_MODE="validator"
```

### 3. Bootnode

填写现网 bootnode：

```bash
BOOTNODES="/ip4/服务器IP/tcp/30333/p2p/12D3KooW..."
```

注意：不要保留 `< >` 这种占位符。

错误示例：

```bash
BOOTNODES="/ip4/<IP>/tcp/<PORT>/p2p/<PEER_ID>"
```

正确示例：

```bash
BOOTNODES="/ip4/80.96.113.85/tcp/30333/p2p/12D3KooWPRvdc8wcwMfFt2RDh3tvZKDqKezPj9vn1oo9m8FuKjdf"
```

### 4. 验证者账户

填写验证者助记词和地址：

```bash
VALIDATOR_MNEMONIC="word1 word2 word3 ... word12"
VALIDATOR_ADDRESS="X..."
```

要求：

```text
助记词必须是 12 或 24 个英文单词
VALIDATOR_ADDRESS 必须是 X 开头
地址必须和助记词匹配
如果要自动注册验证者，地址需要有 NEX 余额
```

### 5. 是否自动注册验证者

如果地址已有余额，并希望脚本自动提交链上注册交易：

```bash
REGISTER_VALIDATOR="true"
```

如果地址还没有余额，建议先设置：

```bash
REGISTER_VALIDATOR="false"
```

等节点同步、账户充值后，再执行：

```bash
bash remote_deploy.sh register
```

这个命令会先检查验证者服务、等待本机 RPC 就绪并等待节点同步完成，再提交注册交易。

## 三、一键开始部署

在本机执行：

```bash
cd /home/XXXX/桌面/nodedeploy
bash remote_deploy.sh start
```

这个命令会自动完成：

```text
1. 把本机 nodedeploy 目录同步到服务器
2. 在服务器创建部署目录
3. 在服务器安装或使用 tmux
4. 在服务器后台启动部署
5. SSH 断开也不会中断部署
```

看到类似提示说明已经开始后台部署：

```text
[完成] 部署已在后台运行。SSH 断开不会中断部署。
[提示] 查看日志：bash deploy_nexus_server.sh logs
[提示] 查看状态：bash deploy_nexus_server.sh status
```

---

## 四、查看部署日志

部署时间可能较长，尤其编译这些组件时：

```text
librocksdb-sys
wasm-opt-sys
nexus-node
```

查看日志：

```bash
bash remote_deploy.sh logs
```

如果看到类似：

```text
Compiling librocksdb-sys ...
Compiling wasm-opt-sys ...
Compiling nexus-node ...
```

说明正在编译，耐心等待即可。

如果 SSH 断开，不用重新部署。重新打开终端后执行：

```bash
cd /home/XXXX/桌面/nodedeploy
bash remote_deploy.sh logs
```

## 五、查看节点是否在运行

执行：

```bash
bash remote_deploy.sh status
```

它会检查：

```text
tmux 后台会话
cargo/rustc 编译进程
nexus-node 进程
节点端口
systemd 服务状态
```

验证者节点服务名是：

```bash
nexus-validator
```

如果显示：

```text
active (running)
```

说明服务正在运行。

如果你部署的是 RPC 节点，还要继续检查公网入口是否可用；localhost 正常不代表公网 HTTPS 已完成。

```bash
curl -s -H 'Content-Type: application/json' \
  -d '{"id":1,"jsonrpc":"2.0","method":"system_health","params":[]}' \
  http://127.0.0.1:9947 | jq

curl -s -H 'Content-Type: application/json' \
  -d '{"id":1,"jsonrpc":"2.0","method":"system_health","params":[]}' \
  https://你的RPC域名 | jq
```

如果本机可用但域名不可用，继续检查：

```bash
systemctl status nexus-rpc nginx --no-pager
nginx -t
journalctl -u nginx -n 100 --no-pager
journalctl -u nexus-rpc -n 100 --no-pager
```

## 六、进入后台部署会话

如果需要进入远程 tmux 会话：

```bash
bash remote_deploy.sh attach
```

进入后，如果想退出 tmux 但不停止部署，按：

```text
Ctrl + B
然后按 D
```

注意：不要直接按 `Ctrl+C`，否则可能中断正在运行的部署命令。

---

## 七、验证者是否需要注册

需要。

验证者节点运行起来，只代表机器已经启动节点。要真正成为链上验证者，还需要链上注册。

脚本支持自动注册：

```bash
REGISTER_VALIDATOR="true"
```

自动注册会尝试执行：

```text
staking.bond
session.setKeys
staking.validate
```

自动注册成功时，日志里会出现类似：

```text
[完成] 验证者注册交易已提交，等待下个 era 生效
```

如果你一开始设置了：

```bash
REGISTER_VALIDATOR="false"
```

后续可以手动触发注册：

```bash
bash remote_deploy.sh register
```

---

## 八、是否设置了开机自启动

已设置。

验证者节点会创建并启用 systemd 服务：

```bash
/etc/systemd/system/nexus-validator.service
```

脚本会执行：

```bash
systemctl enable nexus-validator
```

所以服务器重启后，验证者节点会自动启动。

你可以检查：

```bash
bash remote_deploy.sh status
```

或者远程执行：

```bash
systemctl is-enabled nexus-validator
systemctl status nexus-validator --no-pager
```

如果返回：

```text
enabled
```

说明已设置开机自启动。

## 十、常见问题

### 1. SSH 断开了怎么办？

不用慌。脚本使用 tmux 在服务器后台运行，SSH 断开不会中断部署。

重新连接后执行：

```bash
bash remote_deploy.sh logs
```

或：

```bash
bash remote_deploy.sh status
```

### 2. 一直停在 Compiling 怎么办？

Rust 编译比较慢，尤其是：

```text
librocksdb-sys
wasm-opt-sys
nexus-node
```

先查看是否还有编译进程：

```bash
bash remote_deploy.sh status
```

如果还有 `cargo`、`rustc` 等进程，说明仍在编译。

### 3. 提示 No such file or directory，secret_ed25519 怎么办？

新版脚本已经自动创建网络密钥目录。重新同步并部署：

```bash
bash remote_deploy.sh start
```

### 4. 没有余额能自动注册验证者吗？

不能。

如果验证者地址没有 NEX 余额，请先设置：

```bash
REGISTER_VALIDATOR="false"
```

部署节点。等充值后再运行：

```bash
bash remote_deploy.sh register
```

### 5. 如何重启验证者节点？

远程执行：

```bash
systemctl restart nexus-validator
```

或查看状态：

```bash
systemctl status nexus-validator --no-pager
```

### 6. 如何查看验证者日志？

远程日志文件（实际目录以部署时输出的 `LOG_ROOT` 为准）：

```bash
${LOG_ROOT}/validator.log
```

远程执行：

```bash
tail -f "${LOG_ROOT}/validator.log"
```

## 十一、安全提醒

请务必注意：

```text
不要把 deploy.env 发给别人
不要泄露 VALIDATOR_MNEMONIC
不要把助记词截图发到群里
不要把助记词提交到 GitHub
不要删除 /data/nexus 目录，除非你明确知道后果
```

验证者助记词相当于账户私钥，泄露后资产可能被转走。

---

## 十二、推荐新手操作顺序

完整流程如下：

```bash
cd /home/XXXX/桌面/nodedeploy
nano deploy.env
bash remote_deploy.sh start
bash remote_deploy.sh logs
bash remote_deploy.sh status
```

如果后续需要手动注册验证者：

```bash
bash remote_deploy.sh register
```

脚本会先等待 validator RPC 就绪和同步完成，再提交注册交易。

如果服务器重启后想确认节点是否自动启动：

```bash
bash remote_deploy.sh status
```
